========================================================
[ ROOT CAUSE — DESIGN PROBLEM ]
========================================================

Script crushgear saat ini = Linear Blind Execution.
Semua module dijalankan berurutan tanpa logika kondisional.

5 masalah desain utama:

  1. Tidak ada service-to-exploit mapping
     → Exploit vsftpd, proftpd, IIS FTP dijalankan semua tanpa cek versi banner.

  2. Tidak ada OS detection → payload mismatch
     → windows/x64/meterpreter dipasang ke target Linux (Ubuntu/Nginx).

  3. Tidak ada handler lifecycle management
     → Handler tidak pernah di-kill sebelum exploit berikutnya dijalankan.

  4. Tidak ada dependency chain untuk post-auth modules
     → Dump/enum MySQL dijalankan sebelum auth berhasil.

  5. Tidak ada port state check sebelum load module
     → SMB module tetap jalan meski 445 timeout.
